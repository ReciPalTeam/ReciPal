================================================================================
REPLIT TASK: FATSECRET INGREDIENT LOOKUP — RECIPAL SUPABASE BACKFILL
================================================================================

CRITICAL CONSTRAINTS — READ FIRST
-----------------------------------
1. DO NOT touch, edit, or modify any file in the ReciPal codebase.
2. DO NOT create, update, or delete any application files whatsoever.
3. This task is EXCLUSIVELY a series of FatSecret API calls synced directly
   to Supabase. Nothing else.
4. All writes go to Supabase only: the `ingredients` and `ingredient_nutrients`
   tables, followed by updating `recipe_ingredients.ingredient_id`.
5. Process in batches of 50 canonical lookups at a time to avoid overloading
   the FatSecret API. Wait for each batch to complete and sync to Supabase
   before starting the next.

================================================================================
WHAT NEEDS TO BE DONE
================================================================================

406 unique ingredient names exist in the `recipe_ingredients` table with a NULL
`ingredient_id`. None of these exist in `ingredients` or `ingredient_nutrients`.

For each canonical name (col 3 below), Replit must:

  STEP 1 — FATSECRET CALL
    Search FatSecret using the canonical name (col 3) as the query term.
    Take the best-matching result and record:
      - fatsecret_food_id
      - fatsecret_serving_id
      - metric_serving_unit
      - all available macro/micronutrient values per 100g

  STEP 2 — INSERT into `ingredients`
    INSERT a new row with:
      - ingredient_id  : new UUID
      - canonical_name : the canonical name from col 3
      - fatsecret_food_id, fatsecret_serving_id, metric_serving_unit
      - category       : infer from the ingredient type
    Only insert ONCE per unique canonical — many raw names share the same
    canonical (see DEDUPLICATION section at the bottom of this file).

  STEP 3 — INSERT into `ingredient_nutrients`
    INSERT a new row linked to the ingredient_id from Step 2 with all
    available fields:
      calories_per_100g, protein_per_100g, carbs_per_100g, fat_per_100g,
      saturated_fat_per_100g, polyunsaturated_fat_per_100g,
      monounsaturated_fat_per_100g, trans_fat_per_100g,
      cholesterol_mg_per_100g, sodium_mg_per_100g, potassium_mg_per_100g,
      calcium_mg_per_100g, iron_mg_per_100g, fiber_per_100g, sugar_per_100g,
      added_sugars_per_100g, vitamin_a_mcg_per_100g, vitamin_c_mg_per_100g,
      vitamin_d_mcg_per_100g, cached_at (= now())

  STEP 4 — UPDATE `recipe_ingredients`
    UPDATE recipe_ingredients
    SET ingredient_id = <new uuid from Step 2>
    WHERE name = '<raw name from col 1>'
    AND ingredient_id IS NULL;
    Run this for every raw name variant that maps to this canonical.

  STEP 5 — NEXT BATCH
    After all 50 in the batch are fetched and synced, proceed to the next 50.

================================================================================
BATCHING INSTRUCTIONS
================================================================================

Process in batches of exactly 50 canonical names at a time.
  - Batch 1  : entries 1–50
  - Batch 2  : entries 51–100
  - Batch 3  : entries 101–150
  - Batch 4  : entries 151–200
  - Batch 5  : entries 201–250
  - Batch 6  : entries 251–300
  - Batch 7  : entries 301–350
  - Batch 8  : entries 351–406 (final batch, 56 entries)

Within each batch, deduplicate first — if multiple raw names in the batch share
the same canonical, make only ONE FatSecret call for that canonical, then apply
the ingredient_id to all matching raw name variants.

Do NOT start the next batch until all Supabase writes for the current batch are
confirmed successful.

================================================================================
TYPE KEY
================================================================================

  ALIAS  = Variant/descriptor of a real ingredient. Search FatSecret using the
           canonical name (col 3). Insert under the canonical, not the raw name.

  BRAND  = Branded product. Resolve to the generic equivalent in col 3. Search
           by canonical and store under the generic name.

  COMBO  = "X or Y" recipe-writing artifact. Use the first/primary item in
           col 3 as the canonical. Look up and store only that item.

  NEW    = Genuine gap with no existing DB equivalent. Search FatSecret using
           col 3 as the query. Requires a net-new row in both `ingredients`
           and `ingredient_nutrients`.
