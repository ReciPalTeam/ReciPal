Replit Prompt (App Update): Replace “Trends Summary” with new “Insights” section (Pro = real-time data)

Goal
- Remove the existing “Trends Summary” card on the relevant dashboard/home/analytics screen.
- Replace it with a new “Insights” section that is only shown to Pro users.
- Insights must be generated from the user’s real logged data and update in real time as new entries are logged/edited.

Scope & Behavior
1) UI change
- Delete the “Trends Summary” card component and any related styling/unused code.
- Add an “Insights” section in the same location, visually consistent with existing design system (same spacing, typography scale, card styles).
- The section should render a list of insight cards grouped by category:
  - Consistency Insights
  - Pattern Detection
  - Pace & Projections
  - Nutritional Gaps
  - Behavioral Nudges
  - Weekly Snapshot

2) Pro gating
- Only Pro users see the full Insights section with computed values.
- Non-Pro users see either:
  a) a locked Insights preview (2–3 blurred sample insights) + upgrade CTA, OR
  b) the section hidden entirely (choose whichever matches current paywall pattern).
- Ensure gating uses the app’s existing subscription/auth mechanism (do not hardcode).

3) Data connections (REAL DATA, REAL TIME)
- Connect Insights computations to the app’s actual underlying nutrition/logging data sources (whatever the app uses today: database tables/collections, API endpoints, cached queries, state stores).
- Use the same “source of truth” used for daily totals and macro targets so numbers match everywhere.
- Insights should recompute automatically when:
  - a user logs a meal/food
  - edits or deletes an entry
  - changes goals/targets
  - changes timezone/date boundaries (if applicable)
- Avoid mock data for Pro users. If there is insufficient user data, show a friendly “Not enough data yet” state per category.

4) Time windows & definitions (use these unless the app already has equivalents)
- “last 7 days” = rolling 7 days ending today (based on user timezone).
- “past 2 weeks” = rolling 14 days.
- “last month” = previous 30 days (or calendar month if the app already uses that — keep consistent).
- “hit calorie target” = daily calories within ±X% or ±Y calories of goal (use the same tolerance the app already uses, otherwise default to ±5%).
- “hit all targets” = calories + macros all within tolerance.

Insight Requirements (compute and display from user data)
A) Consistency Insights
- “You hit your calorie target {X} of the last 7 days”
- “Your most consistent macro is {macroName} (within 5% of goal {P}% of days)”
- “You tend to go over on {macroName} on {day(s)}” (detect day-of-week bias)
- “Longest current streak for hitting all targets: {N} days”

B) Pattern Detection
- “Your average calories are {P}% higher on weekends vs. weekdays”
- “You eat {P}% of your daily calories after 7pm”
- “You’ve been trending {N} calories {above/below} your goal over the past 2 weeks”
- “Your protein intake has {increased/decreased} {P}% compared to last month”

C) Pace & Projections
- “You’re averaging a {N} calorie {deficit/surplus} — your goal implies {G}”
- “At this pace, you’ll hit your goal weight by {Date}”
  - Use existing weight-goal model if present; if not, implement a simple projection from avg daily deficit and estimated kcal/lb (and clearly handle missing weigh-ins).

D) Nutritional Gaps
- “You’ve been under your fiber target {X} of the last 7 days”
- “Your fat intake is consistently {P}% over your target”
- “You’re averaging only {N}g protein — your goal is {G}g”

E) Behavioral Nudges
- “You haven’t logged lunch in {X} of the last 5 days — missing data skews your insights”
- “Your best adherence days are {DayRange}”
- “When you eat breakfast, you’re {M}x more likely to hit your protein goal”
- “Days you log all 3 meals, you average {N} fewer calories”
  - These should be driven by meal-tagging/meal-type metadata if available; if not, implement a minimal heuristic using timestamps + meal windows already in app.

F) Weekly Snapshot (compact summary row/cards)
- Average daily calories vs. goal (number + small delta)
- Best and worst day of the week (by adherence or calories vs goal)
- Top 3 most-logged foods
- Adherence percentage (days meeting targets / total days)

Implementation Notes
- Create a dedicated Insights service/module that:
  - pulls the user’s daily totals + targets + meal logs
  - computes all insight metrics in one place
  - returns a typed structure the UI can render
- Ensure performance: memoize/cache per user/date window; don’t recompute heavy queries on every render.
- Add unit tests for the insight calculations (at least: weekend vs weekday, streak, after-7pm %, “most consistent macro”).
- Add empty/insufficient-data states per category.

Deliverable
- PR/commit that removes “Trends Summary”, adds “Insights” section, wires to real data sources, Pro-gates correctly, and updates live with new logs.
