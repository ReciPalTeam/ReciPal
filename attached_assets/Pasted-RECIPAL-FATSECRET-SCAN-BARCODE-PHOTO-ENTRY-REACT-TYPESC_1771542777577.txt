5) BACKEND: ADD TWO ROUTES (REUSE EXISTING FATSECRET AUTH)
============================================================
Find and reuse the existing FatSecret token retrieval + request helper as-is.
Do not create a second auth layer.

5.1 GET /api/fatsecret/barcode
- Query params: barcode, region=US, language=en
- Normalize:
  - if 12 digits (UPC-A) -> prepend "0" to make 13
  - if 13 digits -> pass through
  - else -> return 400 “Invalid barcode length”
- Call FatSecret:
  GET https://platform.fatsecret.com/rest/food/barcode/find-by-id/v2
  with params: barcode, region, language, format=json
- Return JSON. If FatSecret non-200, pass through status + body.

5.2 POST /api/fatsecret/image-recognition
- Body JSON: { image_b64, include_food_data: true, region: "US", language: "en" }
- Call FatSecret:
  POST https://platform.fatsecret.com/rest/image-recognition/v1
- Ensure server JSON body size supports base64 (set a reasonable limit; do not set extremely high).
- Return JSON. If non-200, pass through status + body.

Error handling:
- Minimal user-friendly error strings for UI display.
- No new global notification systems; reuse existing pattern.
