# Replit Agent Prompt — Fix Macro Wizard "Build Muscle" Logic

## Summary

The Macro Wizard currently has 5 steps: Goal → Sex → Height → Weight → Activity Level. When "Build muscle" is selected, the macro output is broken — it uses static percentage splits that produce carb-dominant results (~17% protein, ~66% carbs, ~17% fat). We need to:

1. Add 2 new conditional wizard steps (only shown when goal = "Build muscle")
2. Replace the macro allocation logic for "Build muscle" with protein-first, bodyweight-anchored math
3. Keep TDEE/BMR calculation and activity multipliers completely unchanged

## Task 2: Replace "Build Muscle" Macro Allocation

Find the function that calculates `protein_g`, `carbs_g`, and `fat_g` for the macro results. Search for keywords: `protein`, `carbs`, `fat`, `macro`, `TDEE`, `0.66`, `0.18`, `0.16`, or similar ratio constants.

**Do NOT change the TDEE/BMR calculation or activity multipliers.** Only replace the macro split logic for the "Build muscle" goal.

### New Protein-First Logic (Build Muscle only)

**Inputs:** `calories` (from existing TDEE calc), `weightLb`, `trainingStyle`, `priority`

#### Step 1 — Protein (anchored to bodyweight)

Protein multiplier lookup (g per lb of bodyweight):

```
strength + lean_gain    → 1.1
strength + balanced     → 0.9
strength + performance  → 0.9
mixed    + lean_gain    → 0.9
mixed    + balanced     → 0.85
mixed    + performance  → 0.85
endurance + (any)       → 0.8
```

```
protein_g = weightLb * multiplier
protein_cal = protein_g * 4
```

Minimum floor: protein must never go below `0.8 g/lb`.

#### Step 2 — Fat (percentage-based with gram floor)

```
fat_g_from_pct = (calories * 0.25) / 9
fat_g_min = weightLb * 0.30
fat_g = max(fat_g_from_pct, fat_g_min)
fat_cal = fat_g * 9
```

#### Step 3 — Carbs (remainder)

```
carb_cal = calories - protein_cal - fat_cal
carbs_g = carb_cal / 4
```

#### Step 4 — Safety Clamps

If `carb_cal < 0`:
1. First, reduce fat target from 25% to 20%: `fat_g_from_pct = (calories * 0.20) / 9`, keep the `fat_g_min` floor, recalculate carbs.
2. If still negative, reduce protein toward `0.8 g/lb` minimum, recalculate carbs.
3. Never return negative carbs — clamp `carbs_g` to `>= 0`.

Round all final gram values to whole numbers.
