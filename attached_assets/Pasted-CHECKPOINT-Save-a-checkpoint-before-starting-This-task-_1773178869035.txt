CHECKPOINT: Save a checkpoint before starting. This task updates the serving counter to use the new min_servings column as its floor instead of the base serving count.
---
CONTEXT
The recipes table now has a min_servings column (already populated for all rows). This defines the minimum practical cooking floor for each recipe:
- min_servings = 1: individual portioned items (burgers, tacos, bowls, salads, smoothies, egg dishes, kebabs, pancakes, etc.)
- min_servings = 2: pot/pan meals that need a minimum batch size (soups, stews, curries, pasta, rice dishes, casseroles, etc.)
- min_servings = base servings: recipes that cannot be scaled down (bread, cake, candy, preserves, baked goods)
The serving counter currently uses recipe.servings (base count) as its floor. This task changes the floor to recipe.min_servings, allowing users to scale DOWN below the base serving count where it makes sense.
