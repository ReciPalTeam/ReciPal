## Prompt

I need you to build a mapping system that assigns every ingredient a `pantry_group` (grocery aisle grouping) alongside its existing `category` (swap-ingredient grouping). The 17 ingredient categories power the Swap Ingredient feature ("give me another protein"). The 14 pantry food groups power the Pantry and grocery list ("what aisle is this in"). These are two separate classification systems that coexist. Do Part 1 first, then Part 2. Run full end-to-end testing when complete.

**Acceptance:** Both constants and the mapping are exported from a single shared file. TypeScript types enforce valid values.

#### Task 2: Build the pantry group classification function for Protein and Carb

**What should happen:** The `Protein` and `Carb` categories each split across multiple pantry groups. Build a function `getPantryGroup(ingredientName, category)` that:

1. For categories that have a direct mapping (everything except Protein and Carb), return the mapping from `CATEGORY_TO_PANTRY_GROUP` immediately
2. For **Protein** category, sub-classify based on ingredient name keywords:
   - **"Meat & Seafood"** (default for Protein): chicken, beef, pork, lamb, turkey, duck, goose, veal, venison, bison, bacon, ham, sausage, salami, pepperoni, prosciutto, shrimp, salmon, tuna, cod, tilapia, crab, lobster, clam, mussel, oyster, squid, octopus, anchovy, sardine, mackerel, mahi, snapper, trout, catfish, scallop, crawfish, fish, seafood, meat, steak, ribs, loin, thigh, breast, wing, drumstick, ground beef, ground turkey, ground pork, oxtail, goat, offal, liver, tongue, chorizo
   - **"Dairy & Eggs"**: egg, eggs
   - **"Prepared Foods & Deli"**: tofu, tempeh, seitan, paneer, beyond, impossible, plant-based, veggie burger, veggie patty, deli, lunch meat, hot dog, frankfurter, protein powder

3. For **Carb** category, sub-classify based on ingredient name keywords:
   - **"Pasta, Rice & Grains"** (default for Carb): pasta, spaghetti, penne, fettuccine, linguine, macaroni, rigatoni, orzo, fusilli, rotini, lasagna, noodle, ramen, udon, soba, rice, basmati, jasmine, arborio, risotto, quinoa, couscous, bulgur, barley, farro, millet, oat, oats, oatmeal, polenta, cornmeal, grits, cereal, granola, flour, semolina, lentil, lentils, chickpea, chickpeas, bean, beans, split pea, dal, mung, urad, chana
   - **"Bread & Bakery"**: bread, tortilla, pita, naan, roti, bun, roll, bagel, croissant, biscuit, muffin, wrap, flatbread, focaccia, ciabatta, brioche, sourdough, cornbread, crouton, breadcrumb, panko, waffle, pancake mix, pizza dough, pie crust, taco shell, dumpling wrapper, wonton wrapper, spring roll wrapper, phyllo, puff pastry

4. The function signature should be:
```typescript
export function getPantryGroup(ingredientName: string, category: IngredientCategory): PantryFoodGroup
```

5. Keyword matching should be case-insensitive and use word boundary matching where practical — "rice" should match "jasmine rice" but "rice" in "price" should not match (use `.toLowerCase()` and check for the keyword as a standalone word or at word boundaries)

6. If a Protein ingredient doesn't match any sub-classification keywords, default to "Meat & Seafood"
7. If a Carb ingredient doesn't match any sub-classification keywords, default to "Pasta, Rice & Grains"

**Acceptance:** `getPantryGroup("chicken breast", "Protein")` returns "Meat & Seafood". `getPantryGroup("eggs", "Protein")` returns "Dairy & Eggs". `getPantryGroup("tofu", "Protein")` returns "Prepared Foods & Deli". `getPantryGroup("spaghetti", "Carb")` returns "Pasta, Rice & Grains". `getPantryGroup("tortilla", "Carb")` returns "Bread & Bakery". `getPantryGroup("paprika", "Seasonings")` returns "Spices & Seasonings".

#### Task 3: Add a `pantry_group` column to the `ingredients` table in Supabase

**What should happen:**

   - Type: `TEXT`
   - Nullable: yes (will be backfilled)
2. Do NOT drop or modify any existing columns
3. Do NOT modify the existing `category` column — both columns coexist

**Acceptance:** The `pantry_group` column exists on the `ingredients` table alongside the `category` column. Both are queryable.

#### Task 4: Backfill `pantry_group` for all existing ingredients

**What should happen:**
1. Write a one-time script (e.g., `scripts/backfill-pantry-groups.ts` or `.js`) that:
   a. Fetches ALL ingredients from the `ingredients` table in Supabase
   b. For each ingredient, runs `getPantryGroup(canonical_name, category)` to compute the pantry group
   c. Updates the ingredient's `pantry_group` column with the computed value
   d. Processes in batches of 50-100 to avoid timeouts
2. Log every update: `"[50/1302] 'chicken breast' (Protein) → Meat & Seafood"`
3. At the end, print a summary grouped by pantry group:
   ```
   Pantry Group Distribution:
   Produce: 308
   Meat & Seafood: 280
   Pasta, Rice & Grains: 220
   ...
   ```
4. Handle errors gracefully — if one ingredient fails, log it and continue

**Acceptance:** Every ingredient in Supabase has a non-null `pantry_group` value. Zero nulls remain. The distribution roughly matches these expected counts: Produce ~308, Meat & Seafood ~280, Pasta Rice & Grains ~220, Spices & Seasonings ~246, Oils Sauces & Condiments ~148, Dairy & Eggs ~110, Bread & Bakery ~80, Baking & Sweets ~66, Snacks & Nuts ~53, Prepared Foods & Deli ~50, Canned & Jarred ~55, Beverages & Alcohol ~36, Non-Food ~19.

#### Task 5: Hook pantry group assignment into the ingredient creation pipeline

**What should happen:**
1. Find where new ingredients are created/inserted into the `ingredients` table (likely after FatSecret resolution or during recipe ingestion)
2. Before inserting, compute the pantry group using `getPantryGroup(ingredientName, category)`

**Acceptance:** New ingredients created through the pipeline always have both `category` and `pantry_group` set. No new ingredients ever have a null `pantry_group`.

#### Task 7: Handle the "Frozen" pantry group (cross-cutting)

**What's happening:** "Frozen" is a storage-method-based group, not a food-type-based group. A frozen chicken breast is still Protein → Meat & Seafood. The "Frozen" pantry group works best for items that are *primarily bought frozen* (frozen puff pastry, frozen dumplings, frozen fruit).

**What should happen:**
1. The `getPantryGroup()` function should check if the ingredient name contains the word "frozen" as a prefix/descriptor AND the item is something primarily bought frozen
2. Items that should be classified as "Frozen" include ingredients whose name starts with or contains "frozen" AND are things like: frozen vegetables, frozen fruit, frozen dumplings, frozen pizza, frozen pie crust, frozen puff pastry, frozen dinner rolls, frozen hash browns, frozen french fries
3. Items that should NOT be classified as "Frozen" even if their name contains "frozen": "frozen chicken breast" should still be "Meat & Seafood" (you buy fresh chicken and freeze it yourself), "frozen butter" should still be "Dairy & Eggs"
4. The heuristic: if the ingredient name starts with "frozen" AND the base ingredient (after removing "frozen") would classify as Veggie, Fruit, or Prepared Batters & Doughs, THEN classify as "Frozen". Otherwise, use the normal pantry group classification.
5. This means:
   - "frozen peas" → Frozen ✓ (veggie, primarily bought frozen)
   - "frozen mixed berries" → Frozen ✓ (fruit, primarily bought frozen)
   - "frozen puff pastry sheets" → Frozen ✓ (prepared dough, primarily bought frozen)
   - "frozen chicken breast" → Meat & Seafood ✓ (protein, you freeze it yourself)
   - "frozen butter" → Dairy & Eggs ✓ (dairy, you freeze it yourself)

**Acceptance:** Ingredients with "frozen" in their name get classified into "Frozen" only when they're items primarily sold frozen (veggies, fruit, dough products). Frozen proteins and dairy stay in their normal pantry groups.
