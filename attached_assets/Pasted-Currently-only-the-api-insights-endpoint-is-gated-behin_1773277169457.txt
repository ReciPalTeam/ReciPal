Currently only the /api/insights endpoint is gated behind Pro on the server. 
The following features need proper server-side enforcement. For each protected 
route, check the authenticated user's subscription_tier from the Supabase 
user_profiles table (the single source of truth after Prompt 1 changes). 
If subscription_tier is not 'pro', return HTTP 403 with JSON body:
  { "error": "Pro subscription required", "feature": "<feature_name>" }

════════════════════════════════════════════
FEATURES TO GATE
════════════════════════════════════════════

1. MACRO TRACKING
   Gate all endpoints related to daily macro targets, consumption logging, 
   and macro progress tracking.
   Search server/routes.ts for routes that read or write to:
     - consumptionLogs table
     - Any daily macro tracking or nutrient goal endpoints
   Feature name for error response: "macro_tracking"

2. MEAL PLANNING
   Gate all endpoints related to creating, reading, updating, or deleting 
   meal plans and planner entries.
   Search server/routes.ts for routes that read or write to:
     - weeklyPlans table
     - planDays table
     - planMeals table
   Feature name for error response: "meal_planning"

3. PANTRY DECAY
   Gate all endpoints related to pantry item expiration or decay tracking.
   Search server/routes.ts for routes that read or write to:
     - pantryItems table (specifically decay/expiration-related operations)
   Note: Basic pantry add/remove may remain free if you determine the app 
   needs it for core functionality. Gate only the decay/expiration tracking 
   features.
   Feature name for error response: "pantry_decay"

4. RECIPE SCALING
   Gate all endpoints that return scaled recipe variants.
   Search server/routes.ts for routes that read from:
     - recipe_steps_variants table (via Supabase)
     - recipe_ingredients_variants table (via Supabase)
   These provide pre-computed scaled versions of recipes. The base recipe 
   at its default serving size should remain free.
   Feature name for error response: "recipe_scaling"

5. NUTRITION INSIGHTS (already gated)
   Confirm that /api/insights still returns 403 for free users.
   Feature name for error response: "nutrition_insights"
