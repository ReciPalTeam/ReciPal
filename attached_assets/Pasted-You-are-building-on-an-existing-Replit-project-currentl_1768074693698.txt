Deterministic Macro Calculation Pipeline (must implement locally; no external API)
1) Convert inputs to metric:
- weight_kg, height_cm
2) Compute BMR via Mifflin–St Jeor:
- Male: BMR = 10w + 6.25h - 5a + 5
- Female: BMR = 10w + 6.25h - 5a - 161
3) Convert BMR -> TDEE using activity multipliers:
- sedentary: 1.2
- lightly active: 1.375
- active: 1.55
- very active: 1.725
(If you have “extra active” later, add 1.9; for now optional)
TDEE = BMR * multiplier
4) Goal calories:
- Lose fat: default -15% (allow -10% to -25% guardrail)
- Maintain: 0%
- Build muscle: +10% (guardrail +5% to +15%)
- Improve energy/performance: +0% to +5% (default +0%)
- Eat healthier: 0% (default)
TargetCals = TDEE * (1 + delta)
5) Allocate macros (protein-first with guardrails):
- Protein factor:
  - Lose fat: 2.0 g/kg
  - Maintain: 1.6 g/kg
  - Build muscle: 1.7 g/kg (or 1.6–1.8)
  - Improve energy/performance: 1.6 g/kg
  - Eat healthier: 1.6 g/kg
Protein_g = weight_kg * factor
Protein_cals = Protein_g * 4
- Fat minimum factor default: 0.8 g/kg (guardrail 0.6–1.0)
Fat_g = weight_kg * 0.8
Fat_cals = Fat_g * 9
- Carbs remainder:
Carb_cals = TargetCals - Protein_cals - Fat_cals
If Carb_cals < 0:
  reduce fat down to minimum 0.6 g/kg before reducing protein
Carb_g = Carb_cals / 4
Guardrails:
- Keep macro % within AMDR bounds where possible:
  Protein 10–35%, Fat 20–35%, Carbs 45–65%
If outside, adjust carbs/fat first (not protein) to fit bounds as best-effort.
6) Rounding:
- Round grams to whole numbers
- Recompute calories = 4P + 4C + 9F
- Adjust carbs by +/- 1–3g if needed to align calories

Info tooltip requirement (“(i) how did you come up with these numbers?”)
- Add (i) icon near computed target displays that opens a modal explaining in plain language:
  - we estimate maintenance calories (TDEE) from BMR and activity
  - then adjust for your goal
  - then allocate macros using protein-first rules and standard calorie-per-gram (4/4/9)
Keep it short and readable.
