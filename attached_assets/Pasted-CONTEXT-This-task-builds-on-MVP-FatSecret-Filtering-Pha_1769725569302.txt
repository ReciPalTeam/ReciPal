PART 2 — CUISINE POST-FILTER FUNCTION

Create function:

filterRecipesByCuisine(
  recipes: Recipe[],
  selectedCuisines: string[] | null
): Recipe[]

Rules:
- If selectedCuisines is null or empty → return recipes unchanged
- Lowercase searchable text = recipe.title + recipe.description
- A recipe matches if ANY keyword matches ANY selected cuisine
- Matching logic is OR across cuisines and OR across keywords
- No fuzzy matching
- No scoring
- No weighting

Fail-open behavior:
- If zero recipes match → return original unfiltered list

--------------------------------------------------
PART 3 — RECIPE STORE INTEGRATION

In client/src/lib/recipe-store.ts:

- After recipes are fetched from the server
- Before they are stored or rendered
- Apply cuisine filtering ONLY if:
  - selectedCuisines.length > 0
  - recipes.length > 0

Ensure:
- Cuisine filtering executes AFTER all upstream hard filters
- Cuisine filtering does NOT affect pagination cursors
- Cuisine filtering does NOT trigger additional API calls
