CONTEXT
ReciPal recipe detail pages and MealDetailPopup have a serving counter with minus/plus buttons. Recipes have a base serving count stored in recipes.servings (e.g. 4 for Calentado). Nutrition is stored in recipe_nutrition_totals as per-serving values keyed to that base serving count (e.g. Calentado: 617.2 cal per serving at 4 servings base = 2468.8 cal total).
Three bugs exist:
BUG 1 — Nutrition displays wrong on initial load and reloads to wrong value
On first render the page briefly shows single-serving nutrition (617 cal), then reloads and jumps to full base-serving-count nutrition (2469 cal). This means the component is initializing nutrition incorrectly before correcting itself.
BUG 2 — Increasing servings decreases calories and macros, and returning to base shows single-serving values
The scaling math is inverted. When the user increases servings the calories go down instead of up. When the user returns to base serving count the display shows per-serving values instead of base-total values.
BUG 3 — GPT is not updating ingredient amounts or steps when servings change
The ingredient list and cooking steps are not reflecting scaled quantities when the serving count changes.
---
THE CORRECT NUTRITION LOGIC — implement this exactly
Supabase stores calories_per_serving and a servings count. These represent one portion of the recipe as written at the base serving count.
Total recipe nutrition = calories_per_serving x nutrition_servings (from recipe_nutrition_totals)
Displayed nutrition at any serving count = (calories_per_serving x nutrition_servings) / desired_servings
This means:
- At base servings (e.g. 4): display = (617.2 x 4) / 4 = 617.2 cal per serving
- At 8 servings: display = (617.2 x 4) / 8 = 308.6 cal per serving
- At 2 servings: this is never shown — the floor is base servings
Wait. Re-read the requirement. The user is NOT changing the number of people eating — they are scaling the recipe UP to make MORE food. So the display should show TOTAL calories for the entire scaled batch, not per-person calories.
THE CORRECT DISPLAY MODEL:
- servings counter = how many servings the user wants to make total
- calories shown = total calories for that many servings
- At base (4 servings): show 617.2 x 4 = 2468.8 cal total
- At 8 servings: show 617.2 x 8 = 4937.6 cal total
- The label should read "for X servings" not "per serving"
Scaled macro formula (total for desired_servings):
  displayed_value = calories_per_serving x desired_servings
Apply to calories, protein, carbs, fat.
This always increases as servings increase. This never decreases below base. This is the only correct interpretation.
