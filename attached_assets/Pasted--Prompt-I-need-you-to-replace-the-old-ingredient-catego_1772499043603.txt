### Part 2: Fix FatSecret to Return Smart Alternatives (Not Same-Name Brand Variants)

#### Task 4: Build a smart FatSecret search query strategy

**What's happening:** When the user swaps an ingredient, FatSecret is being searched with the exact ingredient name (e.g., "paprika"). This returns brand variants of the same ingredient (McCormick Paprika, Great Value Paprika, etc.) instead of genuinely different alternative ingredients.

**What should happen:** Build a function `getAlternativeSearchQueries(ingredientName, category)` that generates smarter FatSecret search queries designed to return different ingredients in the same category. The function should:

1. Take the ingredient name and its category (from `classifyIngredient()`)
2. Return an array of 2-3 search queries that find ALTERNATIVES, not variants

**Query strategy by category:**
- **Protein**: If ingredient is "chicken breast", search for "turkey breast", "pork loin", "tofu" — search for other proteins, not brands of chicken breast. Use a mapping of common protein swaps.
- **Seasonings**: If ingredient is "paprika", search for "smoked paprika", "cayenne pepper", "chili powder" — search for related spices in the same flavor family
- **Dairy**: If ingredient is "cheddar cheese", search for "swiss cheese", "gouda cheese", "provolone" — search for other cheeses, not brands of cheddar
- **Sauces & Condiments**: If ingredient is "barbecue sauce", search for "teriyaki sauce", "honey mustard sauce" — related sauces
- **Veggie**: If ingredient is "broccoli", search for "cauliflower", "green beans", "asparagus" — similar vegetables
- **For all categories**: The search queries should target the CATEGORY broadly, not the specific ingredient name

3. A general fallback approach that works for any category: search FatSecret for the **category name itself** (e.g., search "spices seasonings" when swapping a seasoning, search "cheese" when swapping a cheese) to get a variety of results in that food group

**Implementation approach:**
```typescript
function getAlternativeSearchQueries(ingredientName: string, category: IngredientCategory): string[] {
  // Extract the base food type (e.g., "chicken breast" → "chicken", "cheddar cheese" → "cheese")
  // Generate queries that search for OTHER items in the same food group
  // Example for "paprika" (Seasonings):
  //   → ["ground spices", "chili spices", "cayenne"]
  // Example for "chicken breast" (Protein):
  //   → ["turkey breast", "pork chop", "protein meat"]
  // Example for "cheddar cheese" (Dairy):
  //   → ["swiss cheese", "mozzarella", "cheese varieties"]
}
```

**Acceptance:** The function returns 2-3 search queries per ingredient that target genuinely different alternatives in the same food group. It never just returns the original ingredient name as the query.

#### Task 6: Wire it all together in SwapIngredientPopup

**What should happen:** Update the SwapIngredientPopup component (`client/src/components/swap-ingredient-popup.tsx`) to use the new smart search:
