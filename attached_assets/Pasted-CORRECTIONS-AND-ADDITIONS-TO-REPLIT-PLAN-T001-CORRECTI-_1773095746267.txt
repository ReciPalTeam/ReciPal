T006 — Reconcile recipe_ingredients.display_text with normalized amount values
Blocked By: []
Context:
The serving normalization applied to 190 single-serve recipes (sandwiches, burgers, tacos, burritos, etc.)
divided all recipe_ingredients.amount values by 4. However, display_text is a raw scraped string
(e.g. "1 lb ground beef 80 lean") that was not updated and still reflects the original pre-normalization
quantities. This means the ingredients section of the UI will show wrong quantities for all 190 affected
recipes until display_text is reconciled.
Examples of the mismatch:
  display_text: "1 lb ground beef 80 lean"  |  amount: 0.25  |  unit: lb
  display_text: "8 slices rye bread"         |  amount: 2.0   |  unit: slices
  display_text: "8 whole corn tortillas"     |  amount: 2.0   |  unit: whole
The fix:
display_text should be regenerated from amount + unit + the ingredient name portion of the existing
display_text (everything after the original quantity prefix). Since amount is now the correct value,
the display_text should be rebuilt as: {amount} {unit} {ingredient description}.
Details:
1. Identify all recipe_ingredients rows for the 190 normalized recipes:
   SELECT ri.* FROM recipe_ingredients ri
   JOIN recipes r ON r.recipe_id = ri.recipe_id
   WHERE r.servings = 1
   AND r.dish_type IN (the same dish_type list used in the normalization migration)
2. For each row, call GPT-4o with the following prompt to produce a clean human-readable display_text:
   System prompt:
   "You are a recipe ingredient formatter. Given a numeric amount, a unit, and a raw ingredient
   string, return a single clean ingredient line exactly as it would appear in a printed recipe.
   Format the amount as a common fraction or practical measurement (e.g. 1/4, 1/2, 3/4, 1 1/2)
   rather than a decimal. Do not change the ingredient description. Return only the formatted
   ingredient string with no explanation."
   User message:
   "Amount: {amount}
   Unit: {unit}
   Original ingredient text: {display_text}"
   Example input:
   Amount: 0.25
   Unit: lb
   Original ingredient text: 1 lb ground beef 80 lean
   Expected output:
   1/4 lb ground beef 80 lean
3. Update recipe_ingredients.display_text with the GPT result for each row.
4. Process in batches of 100 using the batchProcess utility to avoid rate limits.
5. Expose this as a one-time admin endpoint: POST /api/reconcile-display-text
   Return { updated: number, skipped: number, errors: number }
   This endpoint is called manually — it does not run on app start.
Files: server/routes.ts (new endpoint), server/reconcileDisplayText.ts (new helper)
Acceptance:
- POST /api/reconcile-display-text runs without errors
- Spot-check: Cheeseburger shows "1/4 lb ground beef", "1 bun", "1 slice cheddar cheese"
- Spot-check: Reuben Sandwich shows "2 slices rye bread", "3 oz corned beef", "2 slices Swiss cheese"
- Spot-check: Tacos Al Pastor shows "2 corn tortillas", "5/16 lbs pork shoulder" (or nearest practical: ~1/3 lb)
- No display_text values contain raw decimals like 0.25, 0.125, 0.0625
