CONTEXT
Every recipe in Supabase has two serving-related fields:
- recipes.servings — the base serving count the recipe was written for (e.g. 4)
- recipes.min_servings — the minimum servings this recipe can be scaled down to (e.g. 1, 2, or equal to servings if locked)
The min_servings values are already populated in the DB for all 2,331 recipes. The current implementation incorrectly uses recipe.servings as both the floor and the initial counter value everywhere. It must be changed to use recipe.min_servings as the floor and starting point instead.
Example: Calentado has servings=4 and min_servings=2. The counter must initialize at 2, step 2→4→6→8, and cannot go below 2.
Example: Colombian Breakfast Calentado with Cheese Stuffed Arepas has servings=4 and min_servings=4. Counter initializes at 4, floor is 4, steps 4→8→12.
Example: A burger with servings=1 and min_servings=1. Counter initializes at 1, steps 1→2→3→4.
