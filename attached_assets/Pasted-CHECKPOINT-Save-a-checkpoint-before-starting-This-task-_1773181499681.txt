CONTEXT
The recipes table now has a min_servings column (already populated for all rows). This defines the minimum practical cooking floor for each recipe:
- min_servings = 1: individual portioned items (burgers, tacos, bowls, salads, etc.)
- min_servings = 2: pot/pan meals (soups, stews, curries, pasta, casseroles, etc.)
- min_servings = base servings: cannot be scaled down (bread, cake, candy, etc.)
We need to update the existing onboarding flow and Edit Preferences to collect serving size preference, leftovers preference, and cook session limits from Pro users — then wire those into the Planner so it uses them when auto-populating meal plans.

TASK 3 — Wire Preferences into Planner Auto-Populate Logic
Update the Planner's auto-populate / auto-generate feature to use the new preferences.
3a. Serving Size Filtering
When the planner selects recipes for a meal plan, filter recipes where min_servings <= preferred_serving_size. A recipe is only eligible if it CAN be served at the user's preferred portion size. Calculate nutrition for each recipe at preferred_serving_size (per_serving * preferred_serving_size).
3b. Leftovers Logic — ONLY when allow_leftovers = true
When allow_leftovers is true:
1. After selecting a recipe for a meal slot, check if the recipe's base serving count is greater than preferred_serving_size.
2. If yes, the planner "cooks" at base serving count and assigns preferred_serving_size to the current slot. The remaining servings become available leftovers.
3. Leftover servings can be assigned to future meal slots for the SAME meal type (lunch leftovers go to future lunches, not breakfast).
4. The same recipe can appear at most leftover_tolerance times total across the entire plan.
5. When assigning a leftover, no additional cook session is counted.
6. Spread leftover assignments across different days — do not stack the same meal on consecutive days.
When allow_leftovers is false:
- Every meal slot gets a uniquely selected recipe.
- No recipe appears more than once in the entire plan.
- The planner does NOT consider batch cooking or leftover distribution.
3c. Cook Session Tracking
Track cook sessions per day. A cook session is counted when a recipe is freshly prepared (not a leftover). The planner must respect max_cook_sessions_per_day:
- If the user has more meal slots than max_cook_sessions_per_day, remaining slots MUST be filled by leftovers (requires allow_leftovers = true).
- If allow_leftovers = false and max_cook_sessions_per_day < meals per day, show a warning: "You have [N] meals per day but only allow [M] cook sessions. Enable leftovers to fill the remaining slots, or increase cook sessions."
3d. Feed Filtering
Update the For You feed and any other recipe feeds that serve Pro users. If the user has a preferred_serving_size, filter recipes where min_servings <= preferred_serving_size. Free users or users without a preferred_serving_size set see the unfiltered feed.
3e. Discover Page
The existing serving size filter on the Discover page should use min_servings <= selectedServingSize (not exact match on base servings). This applies to all users — free and Pro.
