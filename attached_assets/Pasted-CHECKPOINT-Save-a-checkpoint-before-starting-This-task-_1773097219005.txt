CHECKPOINT: Save a checkpoint before starting. This task fixes serving counter behavior and recipe detail page image scrolling.

CONTEXT
ReciPal has a serving adjuster (minus/plus buttons) on the recipe detail page and in MealDetailPopup. Each recipe has a base serving size stored in recipes.servings (the canonical minimum — never reducible). The scaling system only scales up from base. Two bugs exist in the current serving counter implementation, plus a separate UI issue with sticky images on recipe detail pages.
---
TASK 1 — Fix serving counter: enforce base serving floor and correct counter direction
File: client/src/pages/recipe/[id].tsx and client/src/components/meal-detail-popup.tsx
Bug 1 — Counter allows reducing below base serving size:
The minus button must be disabled whenever servings === recipe.servings (the base). A user must never be able to reduce below the recipe's stored base serving count. This applies to all recipes regardless of their base value (1, 4, 8, etc.).
Bug 2 — Counter counts in the wrong direction when clicking up:
The plus button is decrementing servings and the minus button is incrementing. Fix the plus handler to add 1 and the minus handler to subtract 1.
Combined fix for both files:
- Plus button onClick: setServings(prev => prev + 1)
- Minus button onClick: setServings(prev => prev - 1)
- Minus button disabled condition: servings <= recipe.servings (base serving count)
- Plus button should have no upper cap beyond reasonable UX (max 20 is acceptable)
After fixing counter direction and floor guard, verify that every serving change correctly triggers the getScaledSteps API call and that the following all update together in sync:
- Cook time (per cook_time_scale_type formula)
- Serving count display
- Calories per serving
- Protein, carbs, fat macros
- Cooking steps instructions and quantities
The linear_batch cook time formula must reflect the correct batch multiplier based on the corrected counter value:
  If desired_servings > source_servings: Math.ceil(desired_servings / source_servings) x cook_time_minutes
  Cook time display must update every time servings changes, not just on first change.

TASK 2 — Remove sticky/fixed image header on all recipe detail pages
File: client/src/pages/recipe/[id].tsx
Currently the hero image at the top of the recipe detail page is fixed or sticky, meaning it does not scroll with the rest of the page content. Users should be able to scroll down and away from the image naturally as they would on any standard page.
Find the image or hero container at the top of the recipe detail page and remove any CSS classes or inline styles that cause it to be fixed or sticky. This includes removing: position: fixed, position: sticky, sticky, fixed, or any equivalent Tailwind classes (e.g. sticky top-0, fixed top-0) applied to the image wrapper or hero section.
The image should remain visible at the top of the page on load but scroll out of view normally as the user scrolls down.
Apply this same fix to any other recipe-related pages or components that use a fixed or sticky image header.
