GOAL
Fix Instacart unit canonicalization so the Instacart payload always uses clean canonical units instead of descriptor-heavy FatSecret units.

CURRENT ISSUE (from Unit Trace logs)
The system correctly parses a base unit but still sends the original descriptor unit as instacartUnitUsed.

Examples currently wrong:
- "cup, chopped" → should be "cup"
- "tsp, ground" → should be "tsp"
- "cup (8 fl oz)" → should be "cup"
- "oz, without shell, raw (yield after cooking)" → should be "oz"
- "Juice of 1 lime (2\" dia)" → should fallback to "each"
- "serving" → must fallback to "each" (never canonical)

The parsed base unit already exists via fatsecret_unit_parsed events.

REQUIRED FIX
Locate the code where Instacart line items are constructed and where
`instacartUnitUsed` is assigned (the same path producing `instacart_lineitem_mapped` trace events).

Replace unit selection logic with this priority order:

1) If fatsecret_unit_parsed.fallbackApplied === true AND fallbackUnit exists:
      instacartUnitUsed = fallbackUnit
      unitIsCanonical = true

2) Else if parsedUnit exists AND parsedUnit is a supported Instacart unit:
      instacartUnitUsed = canonical(parsedUnit)
      unitIsCanonical = true

3) Else if canonicalUnitCandidate exists:
      instacartUnitUsed = canonicalUnitCandidate
      unitIsCanonical = true

4) Else:
      instacartUnitUsed = "each"
      unitIsCanonical = true
      fallbackReason = "unit_unrecognized"

SPECIAL RULES
- "serving" is NEVER canonical → always convert to "each"
- Any unit containing commas or parentheses must NOT be used directly.
- Always prefer parsedUnit over originalUnitDisplay.
- Do NOT change UI behavior or quantity math.
- Only change Instacart mapping logic.

EXPECTED RESULTS (verify via Unit Trace)
After fix, instacart_lineitem_mapped must show:

Cabbage → "cup"
Oregano → "tsp"
Plain Yogurt (Lowfat) → "cup"
Shrimp → "oz"
Fresh Lime Juice → "each"
98% Fat Free Whole Wheat Tortillas → "each"
