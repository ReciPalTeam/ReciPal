We need to separate RECIPE units from PURCHASE units for Instacart checkout.

Current behavior: instacart_unit_canonicalized + checkout payload uses recipe measurements (tsp/tbsp/cup/etc).
Problem: many items (spices, produce) cannot be purchased in those units; they are purchased as each/head/bunch/bag/jar/etc.

Implement:
1) Keep existing recipe fields: recipeQty + recipeUnit (whatever we parse/canonicalize today).
2) Add purchase decision step that outputs purchaseQty + purchaseUnit + purchaseReason.
3) Update instacart_checkout_payload_ready.simplifiedLineItems to use purchaseQty/purchaseUnit, NOT recipe units.
4) Keep recipe measurement stored in the item object for display/cooking.

MVP rules (based on pantryCategory):
- Spices & Seasonings: ALWAYS purchaseUnit="each", purchaseQty=1 (purchaseReason="spice_container")
- Produce: 
  - If ingredientName contains "Cabbage" => purchaseUnit="head", purchaseQty=1
  - If ingredientName contains "Cilantro" => purchaseUnit="bunch", purchaseQty=1
  - Otherwise if recipe unit is cup/tbsp/tsp => purchaseUnit="each", purchaseQty=1 (purchaseReason="produce_countable_fallback")
- Meat & Seafood: prefer weighed units; keep ounce/lb/gram if present.
- Canned & Jarred / Oils Sauces & Condiments / Bread & Bakery / Pasta Rice & Grains / Frozen / Prepared Foods & Deli / Snacks & Nuts:
  - default purchaseUnit="each", purchaseQty=1 unless recipeQty is clearly >1 and unit already countable.

Add logging:
- New event instacart_purchase_unit_decided with: ingredientName, pantryCategory, recipeQty, recipeUnit, purchaseQty, purchaseUnit, purchaseReason.
- In instacart_checkout_payload_ready, include both recipe and purchase fields per line item for verification.

Test using Shrimp Tacos example:
- Cumin should become purchase: 1 each
- Oregano should become purchase: 1 each
- Cabbage should become purchase: 1 head
- Cilantro should become purchase: 1 bunch
- Shrimp stays 4 ounce
- Yogurt can stay 0.25 cup (MVP ok) or be each if needed later
