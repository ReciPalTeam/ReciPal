STANDARDIZE ALL TOAST AND BANNER POPUP DURATIONS TO 2.5 SECONDS
 
All toast notifications, banner popups, snackbars, and temporary alert messages throughout the app need to be hard set to display for exactly 2.5 seconds (2500ms) and then automatically dismiss/leave the screen.
 
────────────────────────────────────────
WHAT TO CHANGE
────────────────────────────────────────
 
Search the entire codebase for every instance of:
 
1. TOAST NOTIFICATIONS
   - Any toast() calls, showToast(), Toast.show(), or similar toast invocations
   - Any toast configuration objects that set a duration/timeout/delay
   - Any toast library configuration (e.g. react-toastify, react-hot-toast, sonner, native toast, etc.)
   - Default toast duration settings in library config or provider setup
 
2. BANNER POPUPS / SNACKBARS / ALERTS
   - Any temporary banner or alert that auto-dismisses
   - Snackbar components with duration/timeout props
   - Any setTimeout() or animation delay used to hide a notification
   - Any autoHideDuration, visibilityTime, displayDuration, or similar props
 
3. ANIMATION/TRANSITION TIMERS
   - Any fade-out, slide-out, or dismiss animation timers tied to notifications
   - Any onDismiss or onClose callbacks triggered by a timer
 
────────────────────────────────────────
THE RULE
────────────────────────────────────────
 
Every single toast, banner, snackbar, and temporary popup in the app must:
- Display for exactly 2500 milliseconds (2.5 seconds)
- Then automatically leave the screen (dismiss, fade out, slide away, etc.)
- No exceptions -- success, error, warning, info toasts all get 2.5 seconds
- If there is a global/default duration config, set it to 2500ms there too
