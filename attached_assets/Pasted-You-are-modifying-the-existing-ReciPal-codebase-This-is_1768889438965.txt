D) TOTALS LOGIC (CRITICAL)
────────────────────────────────────────

We need a robust totals model to support:
- Planner scheduled meals
- Planner cooked meals
- Planner unmarked meals that auto-count after midnight
- Non-planner "logged consumption events" from:
  (1) Instacart checkout completed after “Get Missing Ingredients”
  (2) "Cook Now" clicked when user has all ingredients in pantry

D1) Define meal consumption states for planner entries:
- scheduled (default when placed on calendar)
- cooked (user tapped Cooked)
- removed (not stored; entry deleted)
- autoCounted (system counted it after midnight because it was still unmarked)

D2) Midnight auto-count rule:
- At midnight LOCAL TIME (user timezone), any meal still in state "scheduled" from the previous calendar day becomes "autoCounted" and counts toward totals.
- It counts ONCE.
- If user later taps Cooked on an autoCounted meal, do NOT double count; simply change state from autoCounted -> cooked.
- If user taps Remove on an autoCounted meal, it must be removed and subtracted from totals.

Implement this deterministically:
- Use a daily rollover check (on app open and periodically if app is open) that:
  - checks lastRolloverDate stored per user
  - if date advanced, process previous day’s scheduled meals into autoCounted

D3) What totals include:
Totals shown on planner summary bar and day cards must include:
- All planner meals that are cooked or autoCounted
- PLUS "logged consumption events" created by:
  a) Checkout completed after Get Missing Ingredients -> logs calories/macros to that day and contributes to weekly totals
  b) Cook Now when user had all ingredients -> logs calories/macros to that day and contributes to weekly totals

Important:
- These non-planner logs DO NOT appear as visible planner entries.
- They only affect that day’s calorie total and the weekly totals.

D4) Data model for non-planner logs:
Create a simple model/table/collection such as:
- ConsumptionLog: { id, userId, date, sourceType, recipeId?, calories, protein, carbs, fat, createdAt }
sourceType enum:
- "checkout_logged_recipe"
- "cooknow_logged_recipe"
- "manual_custom_entry" (Pro-only, see section E)

Day totals = sum(planner cooked/autoCounted) + sum(consumption logs for that day)
Week totals = sum(day totals for week)

D5) Ensure macro data availability:
- If recipe nutrition is unknown (demo mode), use best-effort mock nutrition.
- Do not break totals if nutrition is missing; fallback to calories only.
