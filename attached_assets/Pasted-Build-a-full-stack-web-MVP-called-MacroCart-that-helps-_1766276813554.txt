Build a full-stack web MVP called “MacroCart” that helps users (1) calculate daily macros from a short onboarding questionnaire, (2) generate a 7-day meal plan from a seeded recipe database that fits those macros within tolerance, (3) let users refresh individual meals and lock in meals/days, (4) produce an aggregated shopping list (“cart”), (5) estimate weekly savings by matching cart items to seeded store deals, and (6) track lifetime savings.

STACK / TECH
- Next.js (App Router) + TypeScript
- TailwindCSS + shadcn/ui for UI components
- Prisma ORM + SQLite for local development
- Auth: email/password (simple, secure). Store password hashes (bcrypt).
- Use Server Actions (preferred) or API routes for server logic.

NON-GOALS (DO NOT BUILD IN MVP)
- No real grocery store checkout or add-to-cart integrations
- No external recipe API calls
- No barcode scanning
- No advanced pantry depletion math (simple pantry exclusions only)

PAGES / UX REQUIREMENTS
1) /signup, /login
2) /onboarding (multi-step wizard, save progress)
3) /dashboard (this week summary + lifetime savings + link to plan/cart/recipes)
4) /plan (weekly plan view with day cards)
5) /cart (shopping list + store selection + deals + savings breakdown)
6) /recipes (locked recipes list + recipe detail)

ONBOARDING QUESTIONS (store in UserProfile)
- Goal: cut / maintain / bulk
- Sex, age, height, weight
- Activity level: sedentary / light / moderate / very
- Training days per week (0–7)
- Dietary preferences: vegetarian, vegan, pescatarian, keto, gluten-free, dairy-free (checkboxes)
- Allergies/exclusions: nuts, eggs, soy, fish, shellfish, wheat + free-text exclusions
- Meals per day (2–5) and snacks per day (0–3)
- Cooking time preference: quick (<=20 min) or normal
- Budget mode: cheap / normal
- Preferred stores (choose from seeded stores list)
- Pantry staples toggles (“assume I already have”): rice, oats, olive oil, spices, vinegar, hot sauce, etc.

MACRO CALCULATION (store daily targets)
- Use Mifflin–St Jeor to compute BMR
- Compute TDEE with an activity multiplier based on activity level
- Apply goal adjustment:
  - cut: -15% calories
  - maintain: 0%
  - bulk: +10% calories
- Protein target default:
  - cut: 1.0 g/lb bodyweight
  - maintain: 0.8 g/lb
  - bulk: 0.8 g/lb
- Fats default: 25% of calories (grams = fatCalories/9)
- Carbs = remaining calories after protein and fats (grams = carbCalories/4)
- Allow the user to edit/override their final targets on the plan page (save overrides).

MEAL PLAN GENERATION (seeded recipe DB only)
- Generate a 7-day plan.
- Each day includes the exact number of meals and snacks selected.
- Each recipe has macros per serving (calories/protein/carbs/fat). Do NOT compute macros from ingredients in MVP.
- Daily tolerance rules: each day must be within ±10% of:
  - total calories
  - protein grams
  - carbs grams
  - fat grams
- Approach:
  - Create per-slot targets using default distribution:
    - If 3 meals + 1 snack: 25% / 30% / 35% / 10%
    - If 3 meals + 2 snacks: 22% / 28% / 32% / 9% / 9%
    - If 4 meals + 1 snack: 22% / 26% / 26% / 18% / 8%
  - For each slot, pick a recipe that matches the mealType (breakfast/lunch/dinner/snack), passes dietary restrictions, and is closest to that slot target.
  - Validate totals. If out of tolerance, replace the worst-fitting slot and retry up to 80 attempts per day. Use safe fallback recipes if stuck.

REFRESH + LOCK RULES
- In /plan, show day cards with meals/snacks.
- Each meal card has:
  - Refresh button (replaces only that meal)
  - Lock toggle (locked meals cannot be refreshed)
- When refreshing a meal:
  - Replacement must keep the entire day within tolerance.
  - If no replacement found after 50 tries, show an error toast (“No swap found that keeps macros in range. Try unlocking another meal.”).
- When all meals/snacks in a day are locked, show the day card with a green border.

RECIPES
- /recipes lists all locked recipes for the current week.
- Each recipe has ingredients (with amounts) + instructions + macros per serving.

CART / SHOPPING LIST
- Cart is derived from locked meals only.
- Aggregate ingredients across the week:
  - Normalize ingredient names (lowercase, trim, simple singularization)
  - Sum quantities by unit type (grams where possible; otherwise keep unit strings)
- Group items by category (produce/meat/dairy/pantry/frozen/other) based on ingredient.category in the recipe seed data.
- Cart must be editable:
  - remove item
  - adjust quantity
- Exclude pantry staples toggled “already have”.

SEEDED DEALS + SAVINGS
- Seed at least 2–3 stores and ~30 deals.
- Deal model includes: storeId, itemName, sizeLabel, category, regularPrice, salePrice, weekStartDate.
- In /cart, user selects 1 store to price against (default to their preferred store).
- Match cart items to deals using fuzzy string similarity (simple approach is fine).
- For matched items, display regular vs sale price and compute savings.
- Weekly savings = sum(regular - sale) for matched items.
- Persist weekly savings in a SavingsLedger table and show lifetime savings on /dashboard.

EMAIL (MVP: dev-friendly)
- Add a button “Email me this week’s recipes”.
- Use nodemailer with Ethereal (test SMTP) or console output if SMTP is unavailable.
- Email includes: each day’s locked meals, ingredients, and instructions.

DATABASE MODELS (Prisma)
- User (id, email, passwordHash, createdAt)
- UserProfile (userId, onboarding fields, macro targets, overrides, pantry toggles)
- Recipe (id, name, mealType, prepTimeMinutes, tags[], allergens[], macros JSON, ingredients JSON, instructions)
- WeeklyPlan (id, userId, weekStartDate, createdAt)
- PlanDay (id, weeklyPlanId, date)
- PlanMeal (id, planDayId, slotIndex, mealType, recipeId, locked boolean)
- CartItem (id, weeklyPlanId, name, category, quantity, unit, excluded boolean)
- Store (id, name)
- StoreDeal (id, storeId, itemName, sizeLabel, category, regularPrice, salePrice, weekStartDate)
- SavingsLedger (id, userId, weekStartDate, storeId, savingsAmount)

SEED DATA REQUIREMENTS
- Seed 25–40 recipes:
  - mix of breakfast/lunch/dinner/snack
  - include tags for vegetarian/vegan/gluten-free/dairy-free/high-protein/quick/budget
  - include allergens list
  - include ingredient categories for shopping list grouping
- Seed 2–3 stores
- Seed ~30 store deals spanning common ingredients (chicken, rice, oats, yogurt, frozen veg, eggs, etc.)

UI DETAILS
- Use clean, simple UI with shadcn/ui cards, buttons, tabs, toasts.
- Plan page: macro progress bars per day + totals.
- Loading states and empty states everywhere.
- Keep code organized: /lib for macro math and planner logic; /prisma for schema/seed.
