──────────────────────────────────────────
ALLOWED CALL SITE 1 — Recipe Detail: Missing Ingredient Nutrition
──────────────────────────────────────────
Location: server route that serves ingredient data for the Recipe Detail card.

Current behavior: Ingredient data is bundled from the original recipe fetch. 
New behavior: Ingredient data must come from the Supabase tables in this order:
  1. Query recipe_ingredients joined with ingredient_nutrients via ingredient_id
     for this recipe. If all ingredients have nutrient data, return it — 
     do NOT call FatSecret.
  2. For any ingredient rows where ingredient_nutrients has no entry, 
     look up the ingredient's fatsecret_food_id from the Supabase ingredients table.
  3. If fatsecret_food_id exists, call getFoodById(fatsecret_food_id) from FatSecret.
  4. After fetching, write the nutrient result into the Supabase ingredient_nutrients 
     table so it is cached for future requests. Never call FatSecret for the same 
     ingredient twice.

Return ONLY ingredient/nutrition data — never recipe results.
