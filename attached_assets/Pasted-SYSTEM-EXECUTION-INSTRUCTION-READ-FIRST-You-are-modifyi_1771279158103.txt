WHAT TO BUILD
------------------------------------------------------------
A) New Canonicalization Module
Create:
src/utils/instacartUnitCanonicalizer.ts  (or equivalent in this codebase)

Export a single function used by checkout mapping:
canonicalizeForInstacart(input: {
  quantity: number | string | null | undefined,
  unitDisplay: string | null | undefined,
  ingredientName?: string,
  sourceType?: string,
  correlationId?: string
}): {
  instacartQuantity: number | null,
  instacartUnit: string,              // always non-empty
  normalizedFromUnitDisplay: string,   // what we parsed from unitDisplay
  fallbackReason: string | null,
  confidence: "high" | "medium" | "low"
}

Rules:
1) Parse quantity:
   - Accept strings like "1.000", "0.25", "1/2" if they appear anywhere upstream.
   - If parsing fails, set instacartQuantity = null and fallbackReason.
2) Extract a “base unit token” from unitDisplay:
   - Examples: "cup, chopped", "oz, cooked", "tsp, ground", "cup (8 fl oz)",
     "eggplant, unpeeled (approx 1-1/4 lb)" → base token should NOT become that entire phrase.
   - Strategy: take the first word-like token(s) up to comma/paren, normalize whitespace, lowercase.
3) Map to canonical Instacart units using a deterministic mapping table:
   - cup/cups/c → "cup"
   - tablespoon/tablespoons/tb/tbs → "tablespoon"
   - teaspoon/teaspoons/ts/tsp/tspn → "teaspoon"
   - ounce/ounces/oz → "ounce"
   - pound/pounds/lb/lbs → "pound"
   - gram/grams/g/gs → "gram"
   - kilogram/kilograms/kg/kgs → "kilogram"
   - milliliter/millilitre/milliliters/millilitres/ml/mls → "milliliter"
   - liter/litre/liters/litres/l → "liter"
   - pint/pints/pt/pts → "pint"
   - quart/quarts/qt/qts → "quart"
   - gallon/gallons/gal/gals → "gallon"
   - each → "each"
   - bunch/bunches → "bunch"
   - head/heads → "head"
   - can/cans → "can"
   - package/packages → "package"
   - packet → "packet"
   - ears / small ears → "ears" (or "small ears" if it exists in allowed list)
   - small/medium/large (+ abbreviations) → "small" / "medium" / "large"
4) Special compound packaging units:
   If unitDisplay contains patterns like:
   - "fl oz can", "fl oz container", "fl oz jar", "fl oz pouch"
   - "oz bag", "oz can", "oz container"
   - "lb bag", "lb can", "lb container"
   - "per lb"
   then prefer those exact units IF they exist in allowed list.
5) Dangerous/garbage units:
   If unitDisplay is "serving" OR looks like a descriptive phrase ("eggplant, unpeeled ...", "fruit (2\" dia)")
   then fallback to a safe unit:
   - If ingredient seems countable (heuristic: unitDisplay contains "fruit" or "whole" or ingredientName is like lime/tomato/onion/egg):
     use "each" with quantity rounded reasonably (keep decimals if present)
   - Otherwise fallback to "each" (never pass long descriptive phrases to Instacart)
   and set fallbackReason like "unsupported_unit_display" or "serving_unit"
6) Do not change behavior elsewhere:
   This canonicalization function should be called ONLY at Instacart line item construction.
