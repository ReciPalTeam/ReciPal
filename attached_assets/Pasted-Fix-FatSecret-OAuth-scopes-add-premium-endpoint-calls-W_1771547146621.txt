Fix FatSecret OAuth scopes + add premium endpoint calls WITHOUT breaking existing recipe/food calls. Do NOT guess or change image payload formats.

- If the client is not enabled for those scopes, token request fails with 400 invalid_scope, breaking ALL calls (including basic recipes).
- fatsecretCall(params) posts to server.api (or RELAY /fatsecret) and previously worked with basic scope.

GOAL
1) Restore all existing recipe/food calls immediately.
2) Support barcode + image recognition only when enabled.
3) If not enabled, premium features fail gracefully without impacting basic calls.
4) Do NOT modify image recognition payload structure (unknown whether base64/multipart/etc). Only forward what caller provides.

   FATSECRET_SCOPES_BASIC = "basic"
   FATSECRET_SCOPES_FEATURE = "basic barcode image-recognition"

   - Implement getAccessToken(scopeKey: "basic" | "feature"): Promise<string>

       tokenCacheBasic { accessToken, expiresAt }
       tokenCacheFeature { accessToken, expiresAt }

       POST https://oauth.fatsecret.com/connect/token
       Content-Type: application/x-www-form-urlencoded
       Authorization: Basic base64(clientId:clientSecret)
       Body: grant_type=client_credentials&scope=<scope-string>

     If scopeKey === "feature" use FATSECRET_SCOPES_FEATURE
